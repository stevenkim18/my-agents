# 리스크 분석 보고서
생성일: 2026-03-10

## 제품 개요

**Baskit** - Siri 음성 입력, AI 기반 실제 상품 링크 추천, 구매 이력 학습, 공유 리스트를 결합한 장보기 원스톱 앱. 1인 개발, MVP 우선 출시 후 점진적 기능 추가 전략.

---

## 1. 기술적 리스크

### 1-1. SiriKit / App Intents 제약

| 항목 | 내용 |
|------|------|
| 리스크 | Apple SiriKit의 Lists & Notes 도메인은 지원 인텐트가 제한적이며, iOS 버전별 동작 차이가 존재함. Apple은 앱 심사 과정에서 앱의 핵심 기능과 직접 관련이 없는 인텐트 등록을 거부함 |
| 구체적 문제 | "Baskit에 감자 추가해줘" 형태의 커스텀 음성 명령은 SiriKit Lists 도메인이 아닌 App Intents (iOS 16+) 프레임워크로 구현해야 하며, Siri가 앱 이름을 정확히 인식하지 못하거나 오해석하는 오류 발생 가능 |
| 추가 제약 | Apple 가이드라인상 Siri 요청 처리 중 광고 삽입 불가. 어필리에이트 링크 노출 시 심사 리젝 위험 존재 |

### 1-2. 쇼핑몰 API 정책 변경 리스크

| 항목 | 내용 |
|------|------|
| 리스크 | 외부 쇼핑몰 API는 사전 고지 없이 정책 변경, 서비스 종료, 호출량 제한(Rate Limit) 강화가 가능함 |
| 쿠팡 | 쿠팡 Open API는 2024년 8월 API 변경을 단행하였고, Rate Limit 정책을 도입함. 파트너스 링크 생성 API는 승인된 채널(앱 URL)에서만 사용 가능하며, 미등록 채널 사용 시 부정행위로 간주됨 |
| 네이버쇼핑 | 커머스API센터는 판매자/개발사 대상 플랫폼으로, 일반 소비자 향 상품 검색 API는 별도 승인 프로세스 필요. 스마트스토어 API 일부 서비스 종료 전례 있음 |
| 마켓컬리 / 다이소 / SSG | 공개 외부 연동 API를 제공하지 않으며, 웹 스크래핑은 이용약관 위반 및 법적 리스크 수반 |

### 1-3. AI API 비용 리스크

| 항목 | 내용 |
|------|------|
| 리스크 | 구매 이력 학습 및 상품 추천에 LLM API(OpenAI, Claude 등)를 사용할 경우, 사용자 증가 시 API 호출 비용이 선형 이상으로 증가 |
| 시나리오 | MAU 10,000 기준, 1인당 1일 평균 3회 추천 호출 시 월 약 900,000 토큰. GPT-4o 기준 약 월 $270 이상 발생. 무료 사용자가 대다수일 경우 수익보다 비용이 클 수 있음 |

### 1-4. 확장성 병목

| 항목 | 내용 |
|------|------|
| 리스크 | 공유 리스트 실시간 동기화에 Firebase/Supabase Realtime을 사용할 경우, 동시 접속자 급증 시 무료 플랜 한도 초과. Supabase 무료 플랜 50K MAU 제한, Firebase 무료 플랜은 동시 연결 100개 제한(Realtime Database 기준) |
| 추가 위험 | BaaS 벤더 종속성 - 추후 마이그레이션 비용 및 서비스 가격 인상 시 대응 어려움 |

## 2. 법적/규제 리스크

### 2-1. 개인정보보호법 준수 의무

| 항목 | 내용 |
|------|------|
| 리스크 | 구매 이력, 음성 명령 데이터, 쇼핑 패턴은 개인정보보호법상 개인정보에 해당하며, 수집·이용 목적 고지 및 동의 획득 필수 |
| 관련 규정 | 개인정보보호법 제15조(개인정보 수집·이용), 제22조(동의 방법), 제29조(안전성 확보 조치). 2024년 10월 개정 '개인정보의 안전성 확보조치 기준' 적용 |
| 공유 리스트 | 다른 사용자를 초대하는 기능에서 연락처 접근 시 별도 동의 필요. 초대받은 사용자의 구매 이력이 상대방에게 노출될 경우 제3자 제공 동의 필요 |
| AI 학습 | 구매 이력을 AI 추천 학습에 활용할 경우, 자동화된 의사결정 관련 고지 및 프로파일링 동의 요건 검토 필요 |

| 항목 | 내용 |
|------|------|
| 리스크 | 쿠팡 파트너스 이용약관상 앱 활동 채널은 사전 등록이 필수이며, 미등록 채널에서의 링크 생성·노출은 부정행위로 간주되어 계정 정지 및 수익 환수 가능 |

| 대응 필요 | 어필리에이트 링크가 앱 내에서 사파리 또는 외부 앱으로 연결되는 구조임을 심사 노트에 명확히 기술해야 함 |

## 6. 핵심 리스크 상위 5개 대응 전략

### R1. 쇼핑몰 API 정책 변경 / 연동 불가 [Critical]

**대응 방안:**
- MVP 단계에서는 공식 API를 제공하는 쿠팡(파트너스 API)에만 집중하여 리스크를 최소화
- 네이버쇼핑 연동은 Naver Open API의 쇼핑 검색 API(공개 API)를 활용하되, 이용약관 내 허용 범위 내에서만 사용
- 마켓컬리, 다이소 등 API 미제공 플랫폼은 공식 파트너십 협의를 선행하고, 협의 완료 전까지 로드맵에서 제외
- API 응답을 캐싱하여 단일 API 장애 발생 시에도 서비스 연속성 확보 (캐시 TTL 설정)
- 의존 중인 API별 대체 수단을 사전에 식별해 두고 장애 발생 시 즉시 전환할 수 있도록 설계

**조기 경보 지표:** 쇼핑몰 개발자 포털의 공지사항 주간 모니터링, API 에러율 급등(>5%), 특정 쇼핑몰 링크 클릭 후 전환율 급감

---

### R2. 쿠팡 파트너스 어필리에이트 정책 변경 [High]

**대응 방안:**
- 쿠팡 파트너스 외 네이버 쇼핑 어필리에이트 등 복수의 어필리에이트 프로그램을 동시에 가입하여 단일 의존도 분산
- 유료 구독 수익 비중을 초기부터 적극 키워 어필리에이트 수익 의존도를 낮추는 수익 다각화 추진
- 어필리에이트 외에도 앱 내 프리미엄 기능(예: AI 일괄 장보기 목록 생성 등) 판매로 대체 수익 채널 개발
- 파트너스 계정을 앱 URL로 정확히 등록하고, 이용약관 변경 시 즉각 대응할 수 있도록 정책 변경 알림 구독

**조기 경보 지표:** 쿠팡 파트너스 공식 채널 정책 변경 공지, 월별 어필리에이트 수익 전월 대비 30% 이상 감소

---

### R3. 앱스토어 심사 리젝 (어필리에이트 링크 구조) [High]

**대응 방안:**
- 앱 내에서 외부 쇼핑몰로 링크를 여는 것은 "외부 웹사이트로의 단순 연결"임을 심사 노트(App Review Notes)에 명확히 기술
- 어필리에이트 링크가 앱 내 구매(IAP)를 우회하는 구조가 아님을 코드 및 UX 상에서 명확히 분리
- TestFlight를 통해 외부 베타 테스터로 충분한 테스트 후 제출, 잠재적 리젝 사유 사전 파악
- 앱 심사 거부 시 항소(Appeal) 프로세스 준비 및 Apple 개발자 지원 채널 적극 활용
- 유사 어필리에이트 쇼핑 앱(위시버킷, 애플리케이션 등) 심사 통과 사례를 사전 분석하여 구조 참고

**조기 경보 지표:** Beta 리뷰 단계에서 관련 조항 언급, 앱스토어 가이드라인 업데이트 알림

---

### R4. AI API 비용 과다 발생 [High]

**대응 방안:**
- AI 추천 호출을 사용자 요청 시에만 수행하고, 결과를 캐싱하여 동일 쿼리 재호출 최소화 (예: 동일 아이템 추천 24시간 캐싱)
- MVP 단계에서는 LLM 대신 규칙 기반(Rule-based) 추천 엔진으로 시작하고, AI 추천은 유료 구독자 전용 프리미엄 기능으로 단계적 도입
- AI API 월 사용량 한도(Hard Cap)를 설정하고, 초과 시 자동으로 기본 추천(인기 상품)으로 대체하는 Fallback 구조 설계
- 경량 모델(예: GPT-4o mini, Claude Haiku)을 우선 활용하여 토큰 비용 절감
- 월별 AI 비용 대비 유료 구독 수익 비율을 추적하여 비용 효율성 모니터링

**조기 경보 지표:** 월 AI API 비용이 유료 구독 수익의 30% 초과, 사용자 1인당 AI 호출 횟수 예상치 대비 200% 초과

### R5. 대형 플랫폼 유사 기능 내재화 [High]

**대응 방안:**
- 특정 플랫폼에 종속되지 않는 "멀티 쇼핑몰 통합 뷰"를 Baskit만의 핵심 해자로 구축. 개별 쇼핑몰은 절대 제공할 수 없는 크로스 플랫폼 가격 비교를 차별점으로 강화
- 공유 리스트의 소셜 기능(커플/가족 단위의 구매 협업) 강화로 플랫폼이 복제하기 어려운 네트워크 효과 형성

- 경쟁사 동향을 월 1회 이상 정기 분석하고, 시장 변화 시 6주 내 피벗 가능한 기능 백로그를 준비

**조기 경보 지표:** 주요 쇼핑몰 앱의 공유 장바구니 기능 출시 공지, Siri 쇼핑 연동 기능 Apple 특허 출원 공시
