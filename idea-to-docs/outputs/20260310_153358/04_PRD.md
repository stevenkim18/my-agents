## 2. 타겟 유저

### 2-1. 주요 페르소나

#### 페르소나 A — 김지연 (28세, 직장인 / 예비 신혼)

- **직업**: IT 스타트업 마케터
- **거주 형태**: 남자친구와 동거 시작 예정, 현재는 각자 장보기
- **목표**: 남자친구와 함께 장볼 것을 미리 정리해두고, 마트에서 빠짐없이 사오고 싶다. 쿠팡으로 주문하는 것도 편하게 하고 싶다.
- **불편함**:
  - 각자 카카오톡 채팅창에 "이거 사와"라고 보내지만 서로 읽지 않아 누락이 생긴다
  - 마트 가서 뭘 사야 했는지 기억이 안 나 멍하니 서 있는 경험을 반복한다
  - 쿠팡 앱에서 상품을 검색하면 어떤 게 좋은지 고르기 어렵다
- **사용 시나리오**: 퇴근길 지하철에서 "Baskit에 두부 추가해줘"라고 Siri에게 말한다. 집에 도착한 남자친구가 같은 리스트에서 항목을 확인하고 마트에서 장을 본다.

---

#### 페르소나 B — 이준호 (32세, 1인 가구 / 개발자)

- **직업**: 프리랜서 iOS 개발자
- **거주 형태**: 서울 혼자 거주 3년차
- **목표**: 필요한 것을 떠오르는 즉시 기록하고, 온라인 쇼핑 시 제품 선택에 드는 시간을 줄이고 싶다.
- **불편함**:
  - 냉장고 앞에서 "뭐가 없었더라" 하다가 결국 이것저것 사서 중복 구매가 잦다
  - 쿠팡에서 제품 비교하다 30분이 훌쩍 지나간다
  - 애플 리마인더에 적어두는데, 장보기와 업무 메모가 뒤섞여 불편하다
- **사용 시나리오**: 자다가 "아 간장 없었지"라는 생각이 나면 바로 Siri로 추가한다. 주말에 쿠팡 주문할 때 앱에서 추천된 상품 링크를 탭해 바로 결제한다.

---

#### 페르소나 C — 박수미 (41세, 맞벌이 주부)

- **직업**: 초등학교 교사, 맞벌이 부부
- **거주 형태**: 남편, 자녀 2명과 4인 가족 거주
- **목표**: 남편이 마트에 가도 내가 원하는 걸 정확히 사오게 하고 싶다. 어떤 브랜드, 어떤 용량으로 사야 할지 남편이 알아서 확인할 수 있으면 좋겠다.
- **불편함**:
  - 남편에게 장보기 목록을 문자로 보내면 "이게 뭐야" 하며 전화가 온다
  - 쿠팡, 마켓컬리, SSG를 각각 확인해야 해서 어디서 사는 게 나은지 모르겠다
  - 자주 사는 상품을 매번 검색하는 게 번거롭다
- **사용 시나리오**: 아침에 냉장고를 정리하며 부족한 것을 Baskit에 입력한다. 남편이 퇴근길에 앱을 열어 리스트를 확인하고, 추천 상품 링크를 탭해 쿠팡으로 주문한다.

### 2-2. 사용 시나리오

#### 시나리오 1 — 즉흥 장보기 기억

1. 냉장고를 열다가 달걀이 없다는 것을 인지한다
2. "Baskit에 달걀 추가해줘"라고 Siri에게 말한다
3. 앱을 열지 않아도 리스트에 달걀이 추가됐다는 Siri 확인 응답을 받는다
4. 공유 리스트의 경우 파트너 화면에도 즉시 표시된다

#### 시나리오 2 — 쇼핑 중 상품 선택

1. 쿠팡에서 주문하기 전 Baskit을 연다
2. 리스트에서 달걀 항목을 탭하면 쿠팡 파트너스 API 기반 상품 추천 3개가 표시된다
3. 평점, 가격, 배송 정보를 보고 원하는 상품을 탭한다
4. Safari에서 쿠팡 상품 페이지가 열리고 장바구니에 담거나 바로 구매한다

#### 시나리오 3 — 공유 리스트 협업

1. 파트너 A가 Baskit에서 초대 링크를 생성해 카카오톡으로 보낸다
2. 파트너 B가 링크를 탭하면 앱 설치 후 같은 리스트가 자동으로 연결된다
3. A가 아이템을 추가하면 B 화면에 3초 이내 실시간으로 반영된다
4. B가 마트에서 달걀을 체크하면 A 화면에서도 체크 상태가 즉시 반영된다

#### 시나리오 4 — 자주 사는 것 재구매

1. 앱 홈에서 "자주 사는 것" 탭을 연다
2. 최근 3회 이상 구매한 아이템 목록이 구매 빈도 순으로 표시된다
3. 한 번의 탭으로 전체 또는 일부 아이템을 현재 리스트에 추가한다

## 3. 기능 요구사항

### F01. 장보기 리스트 CRUD + 카테고리 자동 분류

**우선순위**: P0 (MVP 필수)

**기능 설명**

사용자가 장볼 아이템을 자유롭게 추가·수정·삭제하고, 구매 완료 시 체크 표시할 수 있다. 입력된 아이템은 규칙 기반 키워드 매핑에 의해 카테고리(식품, 생활용품, 음료, 기타)로 자동 분류되어 시각적으로 그룹화된다.

**입력**

- 텍스트 입력: 아이템 이름 (자유 텍스트, 최대 50자)
- 수량: 선택 입력 (기본값 1)
- 메모: 선택 입력 (브랜드, 용량 등, 최대 100자)

**출력**

- 카테고리별 그룹화된 리스트 뷰
- 각 아이템: 이름, 수량, 메모, 카테고리 아이콘, 체크 상태 표시
- 체크된 아이템은 리스트 하단으로 이동하거나 흐리게 표시

**카테고리 분류 규칙 (초기 키워드 매핑)**

| 카테고리 | 키워드 예시 |
|----------|-----------|
| 식품 | 쌀, 달걀, 두부, 고기, 채소, 과일, 라면, 김치 등 |
| 음료 | 물, 주스, 우유, 커피, 맥주, 음료 등 |
| 생활용품 | 세제, 화장지, 샴푸, 치약, 청소 등 |
| 기타 | 위 분류에 해당하지 않는 모든 항목 |

**예외 처리 및 엣지케이스**

- 아이템 이름이 공백인 경우: 저장 불가, 인라인 에러 메시지 표시
- 아이템 이름 50자 초과 입력: 입력 필드에서 실시간으로 차단
- 카테고리 매핑 불일치: "기타"로 분류 (사용자가 직접 변경 가능)
- 오프라인 상태에서 아이템 추가: 로컬 저장 후 연결 복구 시 자동 동기화
- 동일 아이템 중복 입력: 허용 (수량 통합 기능은 Should Have)

**수용 기준**

- 아이템 추가 → 리스트 반영 응답 시간 1초 이내
- 카테고리 자동 분류 정확도 80% 이상 (주요 식품·생활용품 키워드 기준)
- 1,000개 아이템을 보유한 리스트에서 스크롤 성능 저하 없음 (60fps 유지)

### F02. Siri 음성 입력 (App Intents, iOS 16+)

**우선순위**: P0 (MVP 필수)

**기능 설명**

사용자가 "Baskit에 [아이템] 추가해줘" 형태의 한국어 자연어 명령으로 앱을 열지 않고도 리스트에 아이템을 추가할 수 있다. App Intents 프레임워크 기반으로 구현하며, Siri가 아이템 인식 후 확인 응답을 제공한다.

**지원 발화 패턴**

| 패턴 | 예시 |
|------|------|
| 기본 추가 | "Baskit에 두부 추가해줘" |
| 수량 포함 | "Baskit에 달걀 두 판 추가해줘" |
| 복수 아이템 | "Baskit에 두부랑 간장 추가해줘" |
| 단순 호출 | "Baskit 열어줘" (앱 실행) |

**입력**

- Siri 음성 입력 (한국어)
- 앱이 백그라운드 또는 잠금 상태여도 동작

**출력**

- Siri 응답: "Baskit에 두부를 추가했어요" 또는 "인식하지 못했어요. 다시 말씀해 주세요."
- 앱 리스트에 아이템 즉시 반영 (공유 리스트인 경우 파트너에게도 동기화)

**예외 처리 및 엣지케이스**

- 아이템 이름 인식 실패: Siri가 재시도 요청, 2회 실패 시 앱으로 이동하여 수동 입력 유도
- 인터넷 미연결: Siri 음성 처리는 시스템 레벨에서 이루어지므로 서버 응답이 없을 경우 로컬 큐에 저장 후 동기화
- 로그인 미완료 상태: Siri가 "Baskit에 로그인이 필요해요"라고 안내 후 앱으로 이동
- 수량 인식 오류 (예: "두 판" → "2" 변환 실패): 수량 1로 기본 저장 후 앱 내에서 수정 유도

**수용 기준**

- "Baskit에 [아이템] 추가해줘" 한국어 명령 인식 성공률 90% 이상 (TestFlight 베타 기준)
- 음성 명령 수신 후 아이템 리스트 반영까지 3초 이내
- iOS 16.0 이상 기기에서 정상 동작 확인

**제약 조건**

- App Intents는 iOS 16.0 이상에서만 지원 (iOS 15 이하 사용자는 텍스트 입력만 가능)
- Apple 가이드라인: Siri 처리 중 광고 또는 어필리에이트 링크 노출 불가

### F03. 공유 리스트 실시간 동기화 (2인)

**우선순위**: P0 (MVP 필수)

**기능 설명**

두 사용자가 하나의 리스트를 공유하여 실시간으로 아이템 추가, 수정, 체크 상태를 동기화한다. 초대 링크 방식으로 파트너를 연결하며, 연결 이후에는 별도 조작 없이 변경 사항이 즉시 반영된다.

**공유 리스트 생성 및 초대 흐름**

1. 기존 리스트에서 "공유" 버튼 탭
2. 공유 링크 생성 (유효기간 72시간)
3. iOS 기본 공유 시트로 링크 전송 (카카오톡, 문자 등)
4. 수신자가 링크 탭 → 앱 설치 유도 또는 앱 실행 → Sign in with Apple → 리스트 연결 완료

**입력**

- 리스트 공유 링크 (URL Scheme 기반)
- 각 사용자의 아이템 추가 / 수정 / 삭제 / 체크 액션

**출력**

- 양쪽 사용자 화면에 변경 사항 실시간 반영 (목표: 3초 이내)
- 상대방이 추가한 아이템에는 상대방 이름 또는 이니셜 표시
- 오프라인 복귀 시 누락된 변경 사항 자동 병합

**예외 처리 및 엣지케이스**

- 동시 편집 충돌: 마지막 저장 기준(Last Write Wins) 적용, 삭제된 아이템은 복구 불가 (삭제 전 확인 다이얼로그 표시)
- 초대 링크 만료 (72시간 초과): 재생성 안내
- 수신자가 이미 다른 공유 리스트에 참여 중: MVP에서는 1개의 공유 리스트만 허용, 기존 공유 탈퇴 후 새 공유 참여 유도
- 공유 해제: 각 사용자의 리스트가 독립적으로 분리되어 유지됨 (아이템 데이터 보존)
- 네트워크 단절 중 변경: 로컬 캐시에 기록, 재연결 시 자동 동기화 (Supabase Realtime 기반)

**수용 기준**

- 아이템 추가/체크 후 상대방 화면 반영까지 3초 이내 (4G/LTE 환경 기준)
- 공유 리스트 초대 링크 생성 → 수신자 연결 완료까지 사용자 액션 5단계 이내
- 동시 접속 2인이 각자 아이템을 초당 1개씩 추가해도 데이터 손실 없음

### F04. 쿠팡 파트너스 API 상품 링크 추천

**우선순위**: P0 (MVP 필수)

**기능 설명**

리스트에 추가된 아이템을 탭하면 쿠팡 파트너스 API 기반 상품 검색 결과가 표시된다. 상품명, 가격, 이미지 썸네일, 쿠팡 파트너스 링크를 보여주며, 사용자가 탭 시 Safari에서 해당 쿠팡 상품 페이지로 이동한다.

**입력**

- 아이템 이름 (쿠팡 파트너스 API 키워드 검색어로 사용)
- 사용자 탭 액션 (아이템 상세 진입)

**출력**

- 상품 카드 목록 (최대 3개): 상품 이미지, 상품명, 가격, 쿠팡 로켓배송 여부
- 탭 시 Safari로 쿠팡 파트너스 링크 오픈 (앱 외부 이동)
- 추천 결과 없을 경우: "쿠팡에서 직접 검색하기" 버튼 표시

**예외 처리 및 엣지케이스**

- 쿠팡 API 호출 실패 또는 타임아웃 (2초 초과): "상품을 불러오는 중 오류가 발생했어요. 다시 시도해 주세요." 표시
- 검색 결과 0건: "해당 상품을 찾지 못했어요. 쿠팡에서 직접 검색해 보세요." + 쿠팡 검색 링크
- 쿠팡 파트너스 API Rate Limit 초과: 캐시된 이전 검색 결과 표시 (캐시 TTL 24시간)
- 오프라인 상태: "오프라인 상태입니다. 네트워크 연결 후 상품을 확인해 주세요."
- 동일 아이템 재검색: 24시간 내 동일 키워드는 캐시 결과 반환 (API 호출 절약)

**수용 기준**

- 아이템 탭 후 상품 추천 결과 표시까지 2초 이내 (캐시 미스 기준)
- 쿠팡 파트너스 앱 채널 등록 완료 및 API 승인 상태에서만 기능 활성화
- 상품 링크 탭 시 Safari 외 인앱 브라우저 사용 금지 (앱스토어 가이드라인 준수)

**주의사항**

- 앱 심사 노트에 "어필리에이트 링크는 Safari 외부 연결임"을 명시
- 쿠팡 파트너스 계정에 앱 URL(App Store 링크)을 정확히 등록해야 수수료 집계 가능

### F05. Sign in with Apple (Apple 계정 간편 로그인)

**우선순위**: P0 (MVP 필수)

**기능 설명**

별도 이메일 회원가입 없이 Apple ID로 즉시 로그인한다. 공유 리스트 연동과 구매 이력 저장을 위한 최소한의 사용자 식별에 사용된다.

**입력**

- Apple ID 인증 (Face ID / Touch ID / 패스코드)
- 이름 및 이메일 공개 여부 선택 (Apple 표준 권한 요청)

**출력**

- 앱 내 로그인 완료 → 홈 화면 이동
- 최초 로그인 시 개인정보처리방침 및 이용약관 동의 화면 노출
- 이후 앱 실행 시 자동 로그인 유지 (토큰 기반)

**예외 처리 및 엣지케이스**

- Apple ID 인증 취소: 로그인 화면 유지, 재시도 가능
- 네트워크 오류: "네트워크 연결을 확인해 주세요." 안내 후 재시도
- Apple 서버 장애: 에러 코드와 함께 "잠시 후 다시 시도해 주세요." 안내
- 탈퇴 후 동일 Apple ID 재가입: 기존 데이터는 복구 불가 (탈퇴 전 안내 팝업 표시)

**수용 기준**

- Apple 가이드라인: 소셜 로그인을 제공하는 앱은 Sign in with Apple 필수 지원
- 로그인 성공 후 홈 화면 진입까지 2초 이내
- 개인정보처리방침 및 이용약관 동의 화면은 최초 로그인 시 1회만 표시

### F06. 구매 이력 저장 + "자주 사는 것" 표시

**우선순위**: P0 (MVP 필수)

**기능 설명**

아이템을 체크(구매 완료) 처리할 때마다 구매 이력이 서버에 저장된다. 저장된 이력을 집계하여 자주 구매한 아이템을 빈도 순으로 "자주 사는 것" 탭에 표시하며, 한 번의 탭으로 현재 리스트에 재추가할 수 있다.

**입력**

- 아이템 체크 액션 (구매 완료 처리)
- "자주 사는 것" 탭 진입 액션

**출력**

- 구매 이력 목록: 날짜, 아이템명, 카테고리 (최대 최근 90일 이력 표시)
- "자주 사는 것" 목록: 최근 90일 내 2회 이상 구매한 아이템, 빈도 순 정렬
- 각 아이템 옆 "추가" 버튼으로 현재 리스트에 원클릭 추가

**예외 처리 및 엣지케이스**

- 체크 후 실수로 취소: 체크 해제 가능 (이력 삭제 처리)
- 구매 이력 90일 미만 (신규 사용자): "아직 이력이 없어요. 장보기를 완료하면 자주 사는 것이 표시됩니다." 안내
- 동일 날짜 동일 아이템 중복 체크: 1회로 집계 (중복 카운트 방지)
- 공유 리스트에서 파트너가 체크한 아이템: 각 사용자의 이력에 개별 저장 (프라이버시 보호)

**수용 기준**

- 구매 이력 저장은 체크 액션 직후 즉시 처리
- "자주 사는 것" 목록 로드 속도 1초 이내
- 최소 빈도 임계값: 90일 내 2회 이상 (초기 설정, 추후 조정 가능)

## 5. 제약 사항

### 5-1. 기술적 제약

| 제약 | 내용 |
|------|------|
| iOS 전용 출시 | Siri App Intents는 iOS 전용. Android는 출시 후 6개월 이상 후 검토 |
| App Intents 최소 iOS | iOS 16.0 미만 사용자는 Siri 음성 입력 기능 사용 불가 |
| 쿠팡 파트너스 API | 앱 채널 사전 등록 및 승인 필요. 승인 전 해당 기능 비활성화 |
| 실시간 동기화 | Supabase Realtime 기반. 무료 플랜 동시 연결 200개 제한 → MAU 500명 이상 시 Pro 업그레이드 필요 |
| AI 기능 제외 | MVP 단계에서 LLM 기반 AI 추천 미포함 (비용 리스크). 규칙 기반 분류·추천만 적용 |
| 멀티 쇼핑몰 연동 미포함 | 마켓컬리, SSG, 다이소는 공개 API 미제공. 파트너십 협의 완료 전 연동 불가 |

### 5-2. 비즈니스 제약

| 제약 | 내용 |
|------|------|
| 1인 개발 | 동시 다발적 기능 개발 불가. MVP 핵심 루프 완성에 집중 |
| 쿠팡 파트너스 정책 | 어필리에이트 링크를 Siri 응답·알림에 삽입 불가 (정책 위반) |
| App Store 수수료 | 구독 수익의 30% 애플 수수료 (출시 1년 이후 15%로 감소) |
| 앱스토어 심사 | 외부 결제 유도 구조로 인해 심사 리젝 가능성 존재. 심사 노트 상세 작성 필수 |
| 개인정보보호법 | 구매 이력, 음성 명령 데이터 수집 시 동의 필수. 미준수 시 과태료 부과 가능 |
| 초기 자본 | 클라우드 비용 및 API 비용 초기 자비 투자 필요. 월 운영비 약 5~10만 원 수준 유지 필수 |
