## 4. 유저 여정 맵 (User Journey Map)

신규 사용자(페르소나 A 김지연 기준)가 처음 앱을 열었을 때부터 핵심 가치를 경험하기까지의 단계입니다.

### 단계 1: 발견 (Discovery)

- **상황**: 남자친구로부터 "이거 설치해봐" 하며 Baskit 공유 링크를 카카오톡으로 받음
- **감정**: 반신반의, "또 앱이야?" 하는 가벼운 귀찮음
- **행동**: 링크를 탭하여 App Store 이동 후 설치

---

### 단계 2: 온보딩 (Onboarding)

- **상황**: 앱 최초 실행 후 온보딩 화면 진입
- **감정**: 가입 절차가 복잡하면 이탈하려는 심리
- **행동**: "Apple로 계속하기" 탭 → Face ID 인증 → 개인정보처리방침 동의
- **결과**: 2초 이내로 홈 화면(빈 리스트) 진입
- **디자인 포인트**: 온보딩 3단계 이하, 불필요한 권한 요청 최소화

---

### 단계 3: 최초 핵심 기능 체험 (First Key Action)

- **상황**: 홈 화면에 빈 리스트. "뭔가 추가해 봐야겠다" 생각
- **행동 A (텍스트)**: 텍스트 입력창을 탭하여 "두부" 입력 후 추가
- **행동 B (Siri)**: 홈 화면의 Siri 연동 안내 배너 탭 → "Baskit에 두부 추가해줘" 시도
- **결과**: 두부가 "식품" 카테고리로 분류되어 리스트에 표시됨
- **감정**: "오, 카테고리가 자동으로 나뉘네" 하는 작은 놀라움

---

### 단계 4: 공유 리스트 활성화 (Collaboration Activation)

- **상황**: 남자친구가 보낸 공유 링크로 앱을 설치했으므로 자동으로 공유 리스트 연결 안내 팝업 노출
- **행동**: "리스트 공유 수락" 버튼 탭
- **결과**: 남자친구의 리스트와 연결 완료. 이미 추가된 아이템들이 표시됨
- **감정**: "아, 이게 같이 쓰는 거구나" 하는 이해와 기대

---

### 단계 5: 핵심 루프 첫 완성 (First Loop Completion)

- **상황**: 퇴근길 지하철. 냉장고에 달걀이 없다는 것이 생각남
- **행동**: 잠금 화면에서 Siri 호출 → "Baskit에 달걀 추가해줘"
- **결과**: Siri가 "달걀을 추가했어요" 응답 / 남자친구 화면에 3초 내 실시간 반영
- **감정**: "이게 진짜 편하네. 카카오톡으로 보내는 것보다 훨씬 낫다"

---

### 단계 6: 구매 완료 및 이력 형성 (Purchase & History)

- **상황**: 남자친구가 마트에서 달걀과 두부를 사면서 각 항목 체크
- **결과**: 내 화면에서도 실시간으로 완료 표시 / 구매 이력 자동 저장
- **감정**: "내가 굳이 확인하지 않아도 되네. 신뢰가 생긴다"

---

### 단계 7: 재구매 습관 형성 (Habit Formation)

- **상황**: 다음 주 장보기 준비. "자주 사는 것" 탭 진입
- **결과**: 지난주에 산 달걀, 두부가 표시됨 / "추가" 탭 한 번으로 리스트 자동 채우기
- **감정**: "이게 있으면 매주 처음부터 입력 안 해도 되겠다" → 앱 지속 사용 동기 형성

---

### 핵심 가치 도달 (Aha Moment)

> "퇴근길에 말만 했는데 남자친구 리스트에 바로 떴고, 마트에서 빠짐없이 사왔다."

이 경험이 3회 이상 반복되면 사용자는 Baskit 없이 장보기를 상상하기 어려워집니다.
