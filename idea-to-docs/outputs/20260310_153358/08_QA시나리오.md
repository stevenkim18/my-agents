## 1. 테스트 전략

### 1.1 테스트 범위 및 목표

| 항목 | 내용 |
|------|------|
| 대상 앱 | Baskit iOS 장보기 앱 (MVP v1.0) |
| 최소 지원 OS | iOS 16.0 이상 |
| 테스트 기기 | iPhone SE 2세대 (최소 화면), iPhone 14 Pro (기준 기기), iPhone 15 Pro Max (최대 화면) |
| 테스트 목표 | MVP 핵심 6개 기능의 Happy Path 및 엣지케이스 검증, 성능 기준 충족 여부 확인, 보안 취약점 사전 차단 |
| 범위 외 | Android, iPad Catalyst, iOS 15 이하, 다국어(영어/일본어) |

### 1.2 테스트 유형별 전략

| 테스트 유형 | 범위 | 도구 | 담당 | 목표 커버리지 |
|-----------|------|------|------|-------------|
| 단위 테스트 (Unit) | 카테고리 분류 로직, 구매 이력 집계, 초대 링크 토큰 생성·검증, JWT 갱신 로직 | XCTest (iOS), RSpec (Rails) | 개발자 | 비즈니스 로직 80% 이상 |
| 통합 테스트 (Integration) | Rails API 엔드포인트, Supabase DB 연동, 쿠팡 파트너스 API 프록시, Realtime 이벤트 브로드캐스트 | RSpec + Rails request spec, Supabase Test Helpers | 개발자 | API 레이어 70% 이상 |
| UI 테스트 (E2E) | 주요 유저 플로우 6개 (로그인, 아이템 추가, 공유 초대, 체크, 추천 상품 탭, 자주 사는 것 추가) | XCUITest | 개발자 + QA | 핵심 플로우 100% |
| 수동 테스트 | Siri 음성 인식 품질, 실기기 성능, UI 접근성, 오프라인 시나리오 | 실기기 테스트 | QA | 전 기능 수동 검증 1회 이상 |
| 성능 테스트 | API 응답 시간, 앱 콜드 스타트, 리스트 스크롤 프레임률, 동기화 지연 | Instruments (Xcode), k6 (API) | 개발자 + QA | 성능 기준 전항목 통과 |
| 보안 테스트 | API 인증·인가, API Key 노출 여부, 공유 링크 예측 가능성, HTTPS 강제 여부 | 수동 침투 테스트, Charles Proxy | QA | 체크리스트 전항목 통과 |

### 1.3 테스트 환경

| 환경 | 설명 | 비고 |
|------|------|------|
| 개발(Development) | 로컬 Rails + Supabase 로컬 에뮬레이터 | 단위·통합 테스트 전용 |
| 스테이징(Staging) | Railway Staging 브랜치 + Supabase Staging 프로젝트 | E2E, 성능, 보안 테스트 |
| 프로덕션(Production) | App Store TestFlight | 베타 테스트, 최종 게이팅 확인 |

### 1.4 테스트 도구 목록

| 도구 | 용도 |
|------|------|
| XCTest | iOS 단위 테스트, UI 테스트 |
| XCUITest | iOS E2E 자동화 테스트 |
| RSpec | Rails 단위·통합 테스트 |
| Xcode Instruments | iOS 성능 프로파일링 (CPU, 메모리, 프레임률) |
| k6 | Rails API 부하 테스트 |
| Charles Proxy | 네트워크 요청 인터셉트, SSL 피닝 검증 |
| TestFlight | 베타 배포 및 외부 테스터 피드백 수집 |
| Sentry | 에러 추적 및 성능 모니터링 (스테이징, 프로덕션) |

## 2. 기능 테스트 시나리오

### F01. 장보기 리스트 CRUD + 카테고리 자동 분류

---

```
TC-01: 아이템 추가 및 식품 카테고리 자동 분류
기능: F01 - 장보기 리스트 CRUD + 카테고리 자동 분류
유저 스토리: US-01

사전 조건:
- 로그인 상태
- 홈 화면(리스트 화면)에 있음
- 네트워크 정상 연결 상태

테스트 단계:
1. 하단 또는 상단의 텍스트 입력창을 탭한다
2. "두부"를 입력한다
3. 추가 버튼(또는 키보드 Return)을 탭한다
4. 리스트 화면을 확인한다

기대 결과:
- 1초 이내에 "두부" 항목이 리스트에 나타난다
- "식품" 카테고리 섹션 하위에 배치된다
- 체크 상태는 미완료(unchecked)이다
- 수량은 기본값 1로 표시된다

우선순위: Critical
```

```
TC-02: 아이템 메모 추가 (수정 플로우)
기능: F01 - 장보기 리스트 CRUD + 카테고리 자동 분류
유저 스토리: US-02

사전 조건:
- 리스트에 "두부" 아이템이 존재함
- 네트워크 정상 연결 상태

테스트 단계:
1. 리스트에서 "두부" 아이템을 탭하여 수정 화면으로 진입한다
2. 메모 입력 필드를 탭한다
3. "풀무원 찌개용 300g"을 입력한다
4. 저장 버튼을 탭한다
5. 리스트 화면으로 돌아와 두부 아이템을 확인한다

기대 결과:
- 메모가 저장되어 리스트의 "두부" 항목 아래에 "풀무원 찌개용 300g" 텍스트가 표시된다
- 저장 응답 시간은 1초 이내이다

우선순위: High
```

```
TC-03: 아이템 체크 (구매 완료 처리)
기능: F01 - 장보기 리스트 CRUD + 카테고리 자동 분류
유저 스토리: US-03

사전 조건:
- 리스트에 "달걀", "두부", "세제" 등 3개 이상의 아이템이 존재함
- 모든 아이템이 미완료 상태임

기대 결과:
- "달걀" 항목이 완료 상태(체크 표시)로 변경된다
- 완료 항목은 리스트 하단으로 이동하거나 흐리게(dimmed) 표시된다
- 나머지 미완료 아이템은 상단에 그대로 표시된다
- 상태 변경은 즉시(1초 이내) 반영된다

우선순위: Critical
```

```
우선순위: High
```

### F02. Siri 음성 입력 (App Intents, iOS 16+)

---

```
TC-07: Siri 기본 음성 추가 (잠금 화면 상태)
기능: F02 - Siri 음성 입력
유저 스토리: US-06

사전 조건:
- iOS 16.0 이상 기기
- 로그인 상태
- 앱이 백그라운드 또는 화면 잠금 상태
- 네트워크 정상 연결 상태
- Siri에 Baskit App Intent 등록 완료

테스트 단계:
1. 기기의 잠금 상태에서 사이드 버튼 길게 눌러 Siri 활성화
2. "Baskit에 두부 추가해줘"라고 말한다
3. Siri 응답을 확인한다
4. 앱을 열어 리스트를 확인한다

기대 결과:
- Siri가 "Baskit에 두부를 추가했어요" 형태의 확인 응답을 제공한다
- 음성 명령 후 3초 이내에 리스트에 "두부" 항목이 추가된다
- 카테고리는 "식품"으로 자동 분류된다
- 공유 리스트인 경우 파트너 화면에도 동기화된다

우선순위: Critical
```

```
TC-08: Siri 수량 포함 음성 추가
```

### F03. 공유 리스트 실시간 동기화 (2인)

---

```
TC-10: 공유 리스트 초대 링크 생성 및 파트너 연결
기능: F03 - 공유 리스트 실시간 동기화
유저 스토리: US-09

사전 조건:
- 사용자 A: 로그인 상태, 기존 리스트 1개 존재
- 사용자 B: 앱 설치 및 로그인 상태 (또는 신규 사용자)
- 두 기기 모두 네트워크 정상 연결 상태

테스트 단계:
1. 사용자 A가 리스트 화면에서 "공유" 버튼을 탭한다
2. iOS 공유 시트가 열리면 링크를 복사하거나 메시지로 전송한다
3. 사용자 B가 해당 링크를 탭한다
4. 신규 사용자인 경우 앱 설치 및 Sign in with Apple을 완료한다
5. 공유 수락 화면에서 "수락" 버튼을 탭한다
6. 사용자 B의 화면을 확인한다

기대 결과:
- 총 사용자 액션 5단계 이내로 리스트 연결이 완료된다
- 사용자 B 화면에 사용자 A의 기존 아이템이 모두 표시된다
- 두 사용자 모두 동일 리스트를 공유하는 상태가 된다

우선순위: Critical
```

```
TC-11: 아이템 추가 시 파트너 화면 실시간 반영
```

```
TC-12: 파트너 체크 상태 실시간 반영

유저 스토리: US-11
```

- [ ] iPhone SE 2세대 (소형 화면)에서 전 기능 UI 깨짐 없음
- [ ] iPhone 14 Pro (기준 기기)에서 전 기능 정상 동작
- [ ] iPhone 15 Pro Max (최대 화면)에서 전 기능 UI 깨짐 없음
- [ ] iOS 16.0 기기에서 Siri App Intents 정상 동작
- [ ] iOS 17.0 기기에서 Siri App Intents 정상 동작
- [ ] 다크 모드에서 UI 깨짐 및 색상 대비 문제 없음
- [ ] Reduce Motion 설정 시 애니메이션 비활성화 또는 최소화 확인

### 8.3 Siri 음성 인식 품질 검증 (베타 테스터 피드백 수집)

베타 테스터 20명에게 다음 발화 패턴을 각 5회씩 테스트 요청합니다.

| 발화 패턴 | 목표 성공률 | 측정 방법 |
|----------|-----------|---------|
| "Baskit에 [단음절 아이템] 추가해줘" (예: 쌀, 물) | 90% 이상 | 테스터 체크리스트 기록 |
| "Baskit에 [다음절 아이템] 추가해줘" (예: 두부, 간장) | 90% 이상 | 테스터 체크리스트 기록 |
| "Baskit에 [아이템] [수량] 추가해줘" (예: 달걀 두 판) | 80% 이상 | 테스터 체크리스트 기록 |
| "Baskit에 [아이템]이랑 [아이템] 추가해줘" (복수) | 75% 이상 | 테스터 체크리스트 기록 |

- [ ] 발화 성공률 집계 완료 (목표치 미달 시 개발팀 이슈 등록)

### 8.4 공유 리스트 베타 검증 (2인 1조 테스터)

- [ ] 2인 테스터 조 5쌍이 공유 리스트 생성 및 초대 완료

- [ ] 동기화 지연 3초 이내 달성 여부 체크리스트 기록

### 8.5 베타 피드백 수집 항목

테스터에게 다음 항목에 대한 피드백을 TestFlight 내 피드백 또는 설문으로 수집합니다.

- [ ] 공유 리스트 초대 과정 불편 사항

## 9. 버그 리포트 템플릿

개발팀과의 효과적인 소통을 위한 표준 버그 리포트 양식입니다.

```
---
버그 ID: BUG-{번호} (예: BUG-001)
보고일: YYYY-MM-DD
보고자: {이름}
심각도: Critical / High / Medium / Low

제목: {버그를 한 문장으로 요약}

---
[환경]
기기: {예: iPhone 14 Pro}
iOS 버전: {예: 17.2}
앱 버전: {예: 1.0.0 (Build 12)}
네트워크: {예: Wi-Fi / 4G / 오프라인}
재현율: {예: 100% / 50% / 간헐적}

---
[사전 조건]
- {테스트 전 상태 기술}
- {예: 로그인 상태, 공유 리스트에 아이템 3개 존재}

---
[재현 단계]
1. {행동}
2. {행동}
3. {행동}

---
[실제 결과]
{실제로 발생한 동작을 구체적으로 기술}

---
[기대 결과]
{정상적으로 동작해야 하는 방식을 기술}

---
[첨부 파일]
- 스크린샷: {첨부 여부}
- 화면 녹화: {첨부 여부}
- Sentry 에러 링크: {URL 또는 N/A}
- 기기 로그 (Console.app): {첨부 여부}

---
[추가 정보]
{관련 TC 번호, 재현 빈도 패턴, 우회 방법 등 참고 사항}
---
```

### 심각도 기준

| 심각도 | 정의 | 예시 |
|--------|------|------|
| Critical | 핵심 기능 사용 불가, 앱 크래시, 데이터 손실 | 로그인 불가, 아이템 추가 후 사라짐, 크래시 |
| High | 주요 기능이 비정상 동작하나 우회 방법 있음 | 공유 동기화 지연 10초 이상, Siri 미동작 |
| Medium | 기능은 동작하나 UI 오류 또는 성능 이슈 | 카테고리 분류 오류, 버튼 터치 영역 좁음 |
| Low | 사소한 UI 불일치, 오탈자, 미관상 문제 | 아이콘 색상 불일치, 오탈자 |

## 10. 출시 전 QA 게이팅 기준

App Store 제출 전 모든 기준을 충족해야 합니다.

### 10.1 버그 기준

| 심각도 | 허용 미해결 버그 수 | 비고 |
|--------|------------------|------|
| Critical | 0개 | 1개라도 존재하면 출시 불가 |
| High | 3개 이하 | 우회 방법 문서화 및 다음 버전 수정 계획 명시 |
| Medium | 10개 이하 | 백로그 등록 및 우선순위 지정 |
| Low | 제한 없음 | 백로그 등록 |

|------|------|-------------|
| 비즈니스 로직 단위 테스트 (Rails/iOS) | 80% 이상 | Xcode Coverage Report, SimpleCov (Rails) |
| 핵심 유저 플로우 E2E 테스트 | 6개 핵심 플로우 100% | XCUITest 실행 결과 |
| 수동 기능 테스트 케이스 실행율 | 100% (전 TC 실행 완료) | 테스트 케이스 추적 시트 |
| 수동 기능 테스트 케이스 통과율 | 95% 이상 | 테스트 케이스 추적 시트 |

|------|------|-------------|

| 주요 API 응답 (p99) | 1초 이내 | 출시 블로킹 |
