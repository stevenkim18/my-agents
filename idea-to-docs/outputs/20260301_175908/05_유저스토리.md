### F01: 링크 붙여넣기 저장

---

```
스토리 ID: US-001
페르소나: 김순자 (P01)
스토리: 나는 스마트폰 사용이 서툰 노인 사용자로서,
        링크를 복잡한 조작 없이 저장하기 위해
        클립보드의 링크를 버튼 한 번 눌러 저장하고 싶다.

인수 조건 (Acceptance Criteria):
- Given: 사용자가 카카오톡 등에서 링크를 복사한 상태로 앱 홈 화면에 접속한다
- When: "링크 저장하기" 버튼(큰 버튼, 최소 56px 높이)을 1회 탭한다
- Then: 클립보드의 URL이 자동으로 입력란에 붙여넣어지며,
        "저장했어요!" 확인 메시지가 3초간 화면 상단에 표시된다

우선순위: Must Have
예상 포인트: 3
```

---

```
스토리 ID: US-002
페르소나: 박영수 (P02)
스토리: 나는 즐겨찾기를 직접 정리하려는 자영업자로서,
        오랫동안 보관하던 링크를 한 번에 옮기기 위해
        URL을 직접 입력창에 붙여넣어 저장하고 싶다.

인수 조건 (Acceptance Criteria):
- Given: 사용자가 저장 화면의 URL 입력창에 포커스를 맞춘다
- When: 직접 URL을 붙여넣거나 입력한 뒤 "저장" 버튼을 탭한다
- Then: URL 유효성 검사 후 저장이 완료되며, 저장 목록 상단에 새 링크가 표시된다

우선순위: Must Have
예상 포인트: 2
```

## 3. 엣지케이스 & 예외 시나리오

### F01 — 링크 붙여넣기 저장

| 시나리오 | 상황 | 기대 처리 |
|----------|------|-----------|
| 네트워크 오류 | 저장 요청 중 인터넷 연결이 끊김 | "인터넷 연결을 확인해 주세요" 메시지 표시, 입력값 유지 |
| 빈 클립보드 | 클립보드에 텍스트가 없을 때 버튼 탭 | "복사한 링크가 없어요. 링크를 먼저 복사해 주세요" 안내 |
| 유효하지 않은 URL | 일반 텍스트 또는 깨진 URL이 클립보드에 있음 | "올바른 링크가 아니에요" 메시지 표시 후 수동 입력 유도 |
| 중복 URL | 이미 저장된 링크와 동일한 URL 저장 시도 | "이미 저장된 링크예요. 그래도 저장할까요?" 확인 다이얼로그 |
| 너무 긴 URL | 2,000자 초과 URL | 서버 측 길이 검증 후 "저장할 수 없는 링크예요" 안내 |

| 파싱 타임아웃 | 외부 사이트 응답이 5초 초과 | 기본값 표시 후 백그라운드 재시도 1회 |

| 로그인 필요 사이트 | 접근 제한된 페이지 | URL과 도메인만 표시하며 파싱 실패 안내 없음 (사용자 혼란 방지) |
| HTTPS 미지원 사이트 | HTTP 전용 사이트 | 파싱은 시도하되 카드에 "보안 주의" 배지 표시 |

### F03 — AI 자동 카테고리 분류

| 시나리오 | 상황 | 기대 처리 |
|----------|------|-----------|
| AI 응답 지연 | 분류 API 응답이 3초 초과 | 일단 "기타"로 저장 후 비동기로 카테고리 업데이트 |
| AI 서비스 장애 | 분류 API 완전 실패 | "기타"로 저장, 사용자에게 별도 알림 없음 (나중에 수동 변경 가능) |
| 분류 신뢰도 낮음 | AI 확신도 임계값(70%) 미달 | "기타"로 분류하고 수동 변경 유도 배지 표시 |
| 동시 다중 저장 | 짧은 시간 안에 여러 링크 저장 | 큐(Queue) 방식으로 순차 처리, 각 링크 저장 완료 후 분류 |

---

### F04 — 저장된 링크 목록 보기

| 시나리오 | 상황 | 기대 처리 |
|----------|------|-----------|
| 빈 상태 (Empty State) | 저장된 링크가 없음 | "아직 저장한 링크가 없어요. 링크를 저장해 보세요!" 일러스트 + CTA 버튼 표시 |
| 카테고리 빈 상태 | 선택한 카테고리에 링크 없음 | "이 카테고리에 아직 링크가 없어요" 메시지 표시 |
| 대량 데이터 | 링크가 500개 이상 존재 | 페이지네이션 또는 무한 스크롤로 20개씩 로드 |
| 네트워크 오류 | 목록 로드 중 연결 끊김 | 마지막 캐시 데이터 표시 + "새로고침" 버튼 제공 |
| 외부 앱 미설치 | 유튜브 링크 탭 시 유튜브 앱 미설치 | 기본 브라우저로 폴백(fallback) 열기 |

---

### F05 — 회원가입/로그인

| 시나리오 | 상황 | 기대 처리 |
|----------|------|-----------|
| 소셜 로그인 취소 | 카카오 인증 화면에서 뒤로 가기 | 로그인 화면으로 돌아오며 오류 메시지 없음 |
| 이메일 중복 가입 | 이미 가입된 이메일로 회원가입 시도 | "이미 사용 중인 이메일이에요. 로그인하시겠어요?" 안내 |
| 인증 메일 미수신 | 회원가입 후 인증 메일이 오지 않음 | "메일 재발송" 버튼 제공 (1분 쿨다운) |
| 세션 만료 | 장기 미사용 후 접속 | 자동 로그아웃 후 로그인 화면 이동, "오랫동안 사용하지 않아 로그아웃됐어요" 메시지 |
| 비밀번호 분실 | 로그인 실패 시 비밀번호를 기억 못 함 | "비밀번호를 잊으셨나요?" 링크 → 이메일로 재설정 메일 발송 |

### F07 — 링크 삭제

| 시나리오 | 상황 | 기대 처리 |
|----------|------|-----------|
| 삭제 중 네트워크 오류 | 삭제 요청 전송 후 연결 끊김 | 낙관적 UI로 목록에서 즉시 제거하되, 실패 시 복원 + "삭제에 실패했어요" 메시지 |
| 권한 없는 삭제 시도 | 다른 사용자 링크에 삭제 API 직접 호출 | 서버 RLS로 거부 (403 반환), 클라이언트에 오류 토스트 표시 |
| 전체 삭제 | 모든 링크를 삭제한 경우 | 빈 상태(Empty State) 화면으로 전환 |

## 4. 유저 여정 맵 (User Journey Map)

### 핵심 시나리오: 김순자 (P01) — 첫 번째 링크 저장 경험

```
[단계 1] 앱 진입
  행동: 딸이 설치해준 "링크쏙" 아이콘을 탭한다
  생각: "이게 뭐지? 잘 모르겠는데..."
  감정: 불안 (낯선 앱)
  화면: 스플래시 → 온보딩 1장 ("링크를 쉽게 저장하세요")

[단계 2] 로그인
  행동: 큰 노란색 "카카오로 시작하기" 버튼을 탭한다
  생각: "카카오는 알아. 이건 할 수 있겠다"
  감정: 안심 (익숙한 카카오)
  화면: 카카오 인증 → 자동 완료 → 홈 화면

[단계 3] 빈 화면 확인
  행동: 홈 화면의 빈 상태를 본다
  생각: "저장된 게 없네. 어디에 뭘 눌러야 하지?"
  감정: 당황
  화면: 빈 상태 일러스트 + 큰 "링크 저장하기" CTA 버튼

[단계 4] 링크 복사 (앱 외부)
  행동: 카카오톡으로 돌아가 딸이 보낸 링크를 길게 눌러 복사한다
  생각: "복사는 해봤어. 이제 다시 저 앱 들어가야지"
  감정: 집중
  화면: 카카오톡

[단계 5] 링크 저장
  행동: 링크쏙 앱으로 돌아와 "링크 저장하기" 버튼을 탭한다
  생각: "버튼이 크네. 이거 누르면 되는 거지?"
  감정: 기대
  화면: 클립보드 자동 감지 → "저장했어요!" 확인 메시지

[단계 6] 핵심 가치 경험
  행동: 저장된 링크 카드를 본다 (큰 썸네일, 사이트 제목, 카테고리 배지)
  생각: "어! 이름이랑 사진도 나오네. 나중에 찾기 쉽겠다!"
  감정: 만족, 성취감
  화면: 홈 목록 — 큰 카드 1개 표시

[단계 7] 재방문 의도 형성
  행동: 앱을 닫는다
  생각: "다음에 링크 받으면 여기 저장해야지"
  감정: 신뢰 형성
```

### 핵심 시나리오: 이미숙 (P03) — 카테고리별 정리 경험

```
[단계 1] 목적 인지
  행동: 수업 자료로 쓸 유튜브 링크를 저장하려고 앱을 연다
  생각: "수업 준비 자료 링크 여기 모아두면 편할 것 같아"
  감정: 기대

[단계 2] 링크 저장 + AI 분류
  행동: 유튜브 링크를 붙여넣어 저장한다
  생각: "교육으로 분류됐네! 딱 맞다"
  감정: 긍정적 놀라움
  화면: 카드에 "교육" 배지 자동 표시

[단계 3] 카테고리 탭 탐색
  행동: 상단 "교육" 탭을 탭한다
  생각: "수업 관련 링크만 모아볼 수 있구나"
  감정: 만족
  화면: 교육 카테고리 필터링된 목록

[단계 4] 잘못된 분류 수정
  행동: 뉴스로 잘못 분류된 요리 영상 링크의 카테고리를 "요리"로 변경한다
  생각: "직접 바꿀 수 있어서 다행이네"
  감정: 주도감, 신뢰 유지

[단계 5] 핵심 가치 경험
  행동: 카테고리 탭을 전환하며 뉴스, 교육, 요리 링크를 따로 확인한다
  생각: "이제 링크 찾느라 시간 낭비 안 해도 되겠다!"
  감정: 높은 만족도
```
