## 1. 제품 개요

### 1.1 제품 비전

스마트폰이 서툰 어르신도 받은 링크를 잃어버리지 않고, 언제든 다시 꺼내 볼 수 있게 한다.

### 1.2 제품 목표

| 기간 | 목표 |
|------|------|
| **3개월** | MVP 출시 및 가입자 200명 확보. "저장 → 재열람" 핵심 흐름의 유효성 검증. 1주 재방문율 40% 달성 |
| **6개월** | 가입자 2,000명 달성. 카카오톡 Share Extension 연동으로 링크 저장 흐름 단순화. Should Have 기능 전체 배포. 수익화 모델(가족 플랜) 베타 테스트 시작 |
| **12개월** | MAU 1만 명. 유료 전환율 5%(500명). 가족 플랜 구독 MRR 145만 원 이상. 앱스토어 평점 4.2 이상 유지 |

| 항목 | 내용 |
|------|------|
| **나이 / 직업** | 67세 / 전업주부, 경기도 거주 |
| **스마트폰 사용 수준** | 카카오톡, 유튜브는 매일 사용하지만 앱 설치·설정은 어려움 |

| **링크쏙 사용 시나리오** | 직접 앱을 설치하고 카카오 소셜 로그인으로 가입. 유튜브와 교회 홈페이지 링크를 함께 저장하고 "종교·신앙", "여행·취미" 탭에서 분리해서 봄. 가끔 카테고리가 틀리게 분류되면 직접 변경함 |

### 2.2 사용 시나리오 요약

**핵심 시나리오: 카카오톡으로 받은 링크 저장**

1. 카카오톡 채팅방에서 영상/기사 링크를 받는다
2. 링크를 길게 눌러 URL을 복사한다
3. 링크쏙 앱을 연다
4. 화면 중앙의 큰 "붙여넣기" 버튼을 누른다
5. "저장했어요!" 메시지를 확인하고 앱을 닫는다
6. 나중에 앱을 열어 카테고리 탭에서 저장한 링크를 찾아 시청한다

## 3. 기능 요구사항

### MVP Must Have 기능 (7개)

---

#### F01. 링크 붙여넣기 저장

**기능 설명 (사용자 관점)**
클립보드에 복사한 URL을 붙여넣기 버튼 하나로 저장한다. 유튜브 링크, 일반 웹사이트 URL 모두 지원한다. 저장이 완료되면 큰 글씨로 "저장했어요!" 피드백을 보여준다.

| 항목 | 내용 |
|------|------|
| **입력** | 클립보드의 URL 문자열 (유튜브, 일반 웹사이트) |
| **출력** | 저장 완료 피드백 메시지 ("저장했어요!", 큰 글씨 + 확인 버튼). Supabase `links` 테이블에 레코드 생성 |
| **동작 방식** | 앱 실행 시 클립보드에 URL이 감지되면 자동으로 입력창에 채워줌 (사용자 확인 후 저장). 수동으로 붙여넣기 버튼을 눌러 저장하는 방식도 동시 지원 |
| **우선순위** | P0 (출시 필수) |

**예외 처리 및 엣지케이스**
- 유효하지 않은 URL(http/https 미포함, 텍스트 문자열): "올바른 링크가 아닙니다" 안내 메시지 표시 후 저장 중단
- 중복 URL 저장 시도: "이미 저장한 링크입니다. 다시 저장할까요?" 확인 다이얼로그 표시
- 네트워크 오프라인 상태: "인터넷 연결을 확인해주세요" 안내. URL 자체는 임시 저장 후 온라인 복구 시 자동 동기화 (또는 저장 실패 명확히 안내)
- 클립보드 접근 권한 미허용: 수동 붙여넣기 입력창으로 대체 안내

#### F02. URL 메타데이터 자동 파싱

**기능 설명 (사용자 관점)**
링크를 저장하면 자동으로 해당 콘텐츠의 제목과 썸네일 이미지를 가져온다. 사용자가 아무것도 입력하지 않아도 어떤 링크인지 바로 알아볼 수 있다.

| 항목 | 내용 |
|------|------|
| **입력** | 저장된 URL |
| **출력** | 콘텐츠 제목 (최대 60자), 썸네일 이미지 URL, 도메인 정보 |
| **유튜브 처리** | YouTube oEmbed API 1차 시도 → YouTube Data API v3 fallback → 실패 시 기본 이미지 |
| **일반 웹사이트 처리** | Open Graph (`og:title`, `og:image`) 파싱 → `<title>` 태그 fallback → 실패 시 URL 도메인명 표시 |
| **처리 방식** | 서버사이드(Next.js Server Action 또는 API Route)에서 CORS 우회 처리 |
| **우선순위** | P0 (출시 필수) |

**예외 처리 및 엣지케이스**
- 파싱 실패(CORS, 봇 차단, 타임아웃): 기본 플레이스홀더 이미지 + URL 도메인명 표시. 저장 자체는 항상 성공
- 파싱 응답 지연(3초 이상): 로딩 인디케이터 표시 후 완료 시 업데이트
- 이미지 URL이 만료되거나 깨진 경우: 기본 플레이스홀더 이미지로 자동 대체
- 유튜브 Shorts, 재생목록 URL: 일반 유튜브 영상과 동일하게 처리 (딥링크만 제공)

#### F03. AI 자동 카테고리 분류

**기능 설명 (사용자 관점)**
링크를 저장하면 AI가 자동으로 콘텐츠의 주제를 파악해 적절한 카테고리에 넣어준다. 직접 분류할 필요 없이 저장만 하면 된다. 분류 결과가 마음에 들지 않으면 직접 바꿀 수 있다.

| 항목 | 내용 |
|------|------|
| **입력** | URL + 파싱된 제목 + 도메인 정보 |
| **출력** | 카테고리 레이블 (7개 중 1개) |
| **카테고리 목록** | 건강, 요리·음식, 뉴스·시사, 종교·신앙, 여행·취미, 가족·생활, 기타 |
| **처리 방식** | 비동기 처리 (저장 즉시 "분류 중..." 표시 → 완료 시 카테고리 뱃지 업데이트) |
| **AI 모델** | gpt-4o-mini 또는 Claude Haiku (한국어 카테고리 분류 최적화, 비용 우선) |
| **수동 변경** | 저장 목록에서 카테고리 변경 가능 (단순 드롭다운, 7개 옵션) |
| **우선순위** | P0 (출시 필수) |

**예외 처리 및 엣지케이스**
- AI API 응답 실패(타임아웃, 서버 오류): "기타"로 자동 배정, 저장 실패 없음
- 카테고리 판단 불명확(신뢰도 낮음): "기타"로 배정 후 사용자가 변경 가능
- AI API 비용 한도 초과: 해당 기간 동안 모든 신규 저장 링크는 "기타"로 배정, 비용 한도 초과 알림 운영자에게 발송
- 프롬프트 인젝션 방지: URL과 제목만 전달하며 본문 전체 크롤링 없음

#### F04. 저장된 링크 목록 보기

**기능 설명 (사용자 관점)**
저장한 링크들을 큰 썸네일과 큰 글씨로 한눈에 볼 수 있다. 카테고리 탭으로 원하는 종류만 골라 볼 수 있으며, 링크를 누르면 유튜브 앱이나 브라우저가 열린다.

| 항목 | 내용 |
|------|------|
| **입력** | 로그인된 사용자의 저장 링크 전체 |
| **출력** | 링크 카드 목록 (썸네일 + 제목 + 카테고리 뱃지 + 저장 날짜) |
| **폰트 크기** | 최소 18px 이상. 제목 텍스트 20px 권장 |
| **썸네일 크기** | 최소 120 × 90px. 카드형 레이아웃에서는 전체 너비의 100% 이미지 권장 |
| **카테고리 탭** | "전체" 포함 8개 탭. 탭 버튼 최소 높이 44px. 가로 스크롤 지원 |
| **정렬 기준** | 기본: 저장 최신순 |
| **딥링크** | 유튜브 링크: 유튜브 앱으로 이동 (앱 미설치 시 모바일 브라우저). 일반 링크: 기본 브라우저로 이동 |
| **우선순위** | P0 (출시 필수) |

**예외 처리 및 엣지케이스**
- 저장된 링크 없음: "아직 저장한 링크가 없어요. 링크를 붙여넣어 저장해보세요!" 빈 상태 안내
- 썸네일 로딩 실패: 기본 플레이스홀더 이미지 표시 (깨진 이미지 아이콘 방지)
- 분류 진행 중인 링크: "분류 중..." 뱃지 표시 후 완료 시 자동 갱신
- 카테고리 탭 항목이 비어 있는 경우: 해당 탭 선택 시 "이 카테고리에 저장된 링크가 없어요" 안내
- 링크 재열람 시 원본 콘텐츠 삭제됨: 브라우저/앱이 열리지 않거나 오류 페이지 표시. 링크쏙에서 별도 처리 불가 (안내 없음)

#### F05. 회원가입 및 로그인

**기능 설명 (사용자 관점)**
이메일과 비밀번호로 회원가입하거나, 카카오 계정으로 간편하게 시작할 수 있다. 로그인하면 어떤 기기에서도 저장한 링크를 볼 수 있다.

| 항목 | 내용 |
|------|------|
| **입력** | 이메일 + 비밀번호 (이메일 인증 포함) 또는 카카오 소셜 로그인 |
| **출력** | 인증된 세션 토큰 (Supabase Auth JWT) |
| **인증 방식** | Supabase Auth (이메일 + 카카오 OAuth 2.0) |
| **비밀번호 규칙** | 최소 8자, 영문+숫자 조합. 입력 난이도 안내 문구 제공 |
| **세션 유지** | "로그인 유지" 기본 활성화 (재로그인 불편 최소화) |
| **우선순위** | P0 (출시 필수) |

**예외 처리 및 엣지케이스**
- 이미 가입된 이메일로 중복 가입 시도: "이미 가입된 이메일입니다. 로그인해주세요" 안내
- 비밀번호 분실: "비밀번호 찾기" 이메일 발송 (Supabase 기본 제공)
- 카카오 로그인 중 네트워크 오류: 재시도 버튼 표시

#### F06. 기본 보안 설정

**기능 설명 (사용자 관점)**
나의 저장 링크는 나만 볼 수 있다. 다른 사람이 내 저장 내역에 접근할 수 없다.

**예외 처리 및 엣지케이스**
