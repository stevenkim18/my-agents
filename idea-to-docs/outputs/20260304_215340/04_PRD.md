# 제품 요구사항 문서 (PRD)
버전: 0.1 | 생성일: 2026-03-04

## 1. 제품 개요

### 제품 비전

커플과 가족이 일정을 따로 묻지 않아도 서로의 하루를 함께 알 수 있는, 가장 자연스러운 공유 캘린더를 만든다.

### 제품 목표

| 기간 | 목표 |
|------|------|
| **3개월** | iOS MVP 출시 및 클로즈드 베타 → 오픈 베타 전환. 활성 그룹 50쌍 이상 확보, 앱스토어 평점 4.0 이상 달성 |
| **6개월** | 7일 리텐션 40% 달성, 그룹 연결 완료율 60% 달성, Google Calendar 연동(v1.1) 출시, 누적 다운로드 5,000건 이상 |
| **12개월** | 유료 구독 전환 사용자 1,000명 이상, 누적 다운로드 5만 건 이상, 연 매출 3,500만 원 이상 달성 |

### 성공 지표 (KPI)

| 지표 | 목표값 | 측정 기준 |
|------|--------|-----------|
| 7일 리텐션 | 40% 이상 | 신규 설치 후 7일째 재방문 비율 |
| 그룹 연결 완료율 | 60% 이상 | 앱 설치 후 48시간 이내 파트너 연결 완료 비율 |
| 공동 일정 초대 수락율 | 70% 이상 | 초대 발송 대비 수락 비율 |
| 1인당 주간 일정 생성 수 | 평균 3건 이상 | 활성 사용자 기준 주간 이벤트 생성 수 |
| 앱스토어 평점 | 4.0 이상 | 누적 100리뷰 기준 |
| 유료 전환율 (12개월) | 2% 이상 | 누적 설치 대비 유료 구독 전환 비율 |

## 2. 타겟 유저

### 페르소나 1 — 바쁜 맞벌이 커플 "지수 & 민준"

| 항목 | 내용 |
|------|------|
| 이름 | 박지수 (30세), 김민준 (32세) |
| 직업 | 마케터 / IT 회사 개발자 |
| 생활 방식 | 동거 중, 각자 업무 일정이 불규칙하여 저녁 약속, 주말 계획을 매번 카카오톡으로 맞춤 |
| 목표 | "오늘 몇 시에 들어와?" 같은 대화를 줄이고 상대 일정을 한눈에 파악하고 싶다 |
| 불편함 | 일정 변경이 생겨도 카카오톡을 보내지 않으면 상대가 모른다. Google Calendar와 Apple Calendar를 각자 쓰기 때문에 서로의 일정이 보이지 않는다 |
| iOS 사용 여부 | 둘 다 iPhone 사용자 |

**사용 시나리오**

지수가 갑자기 야근이 생겨 저녁 약속을 취소해야 할 때, WeCal에서 해당 일정을 수정하면 민준의 캘린더에 즉시 반영되고 푸시 알림이 발송된다. 민준은 별도 연락 없이 변경 사실을 확인한다.

### 페르소나 2 — 장거리 연애 중인 "하은 & 준서"

| 항목 | 내용 |
|------|------|
| 이름 | 이하은 (27세), 정준서 (28세) |
| 직업 | 간호사 / 대학원생 |
| 생활 방식 | 서울 ↔ 부산 장거리 연애. 하은은 3교대 근무로 일정이 복잡하고, 준서는 논문 스케줄이 불규칙 |
| 목표 | 다음에 볼 수 있는 날짜를 빠르게 파악하고 공동 약속을 잡고 싶다 |
| 불편함 | 통화나 문자로 "그 주에 뭐 있어?"를 반복한다. 날짜를 정해도 나중에 한 명이 일정을 잊거나 겹치는 경우가 생긴다 |
| iOS 사용 여부 | 하은은 iPhone, 준서는 안드로이드 → MVP에서는 iPhone 사용자만 사용 가능 (준서에게 iPhone 지인을 통해 초대 링크 공유 가능) |

**사용 시나리오**

준서가 다음 달 방문 계획을 WeCal에 공동 일정으로 초대하면 하은이 수락/거절을 탭 한 번으로 응답한다. 수락 시 양측 캘린더에 자동으로 일정이 추가되고 하루 전 리마인더가 발송된다.

### 페르소나 3 — 자녀 일정 관리가 필요한 맞벌이 부부 "유진 & 상훈"

| 항목 | 내용 |
|------|------|
| 이름 | 최유진 (36세), 박상훈 (38세) |
| 직업 | 프리랜서 디자이너 / 영업직 |
| 생활 방식 | 초등학교 1학년 자녀 1명. 학교 행사, 병원 예약, 가족 여행 일정을 분담하여 관리 |
| 목표 | "오늘 애 학교 행사 있다고 했지?" 같은 확인 대화 없이 가족 일정을 한 곳에서 관리하고 싶다 |
| 불편함 | 상훈이 외근 중일 때 일정 변경을 인지하지 못해 픽업을 놓치는 등 실수가 반복된다. 기존 범용 캘린더는 공유 기능이 번거롭고 알림이 자신에게만 와서 공동 관리가 안 된다 |
| iOS 사용 여부 | 둘 다 iPhone 사용자 |

**사용 시나리오**

유진이 WeCal에 "소아과 예약 — 오후 4시" 일정을 추가하고 상훈을 공동 참석자로 초대한다. 상훈이 수락하면 양쪽 캘린더에 일정이 표시되고, 당일 오전 리마인더 알림이 상훈의 폰에도 발송된다.

## 3. 기능 요구사항

### F01. 회원가입 및 로그인

**기능 설명 (사용자 관점)**
앱 최초 실행 시 Sign in with Apple로 간편하게 계정을 생성하고 로그인한다. 기존 사용자는 자동으로 로그인 상태가 유지된다.

**입력 / 출력**
- 입력: Apple ID 인증 토큰
- 출력: 사용자 계정 생성 및 JWT 세션 토큰 발급, 온보딩 화면으로 이동

**예외 처리 및 엣지케이스**
- Apple ID 인증 실패 시 에러 메시지 표시 및 재시도 버튼 제공
- 기기 변경 후 동일 Apple ID 로그인 시 기존 계정 복원
- 네트워크 오프라인 상태에서는 로그인 불가 안내 메시지 표시
- 사용자가 이름 제공을 거부한 경우 기본 닉네임("사용자_숫자") 자동 부여

**우선순위**: 최우선 (P0)

---

### F02. 그룹 생성 및 초대

**기능 설명 (사용자 관점)**
회원가입 직후 또는 설정에서 그룹을 생성한다. 초대 링크 또는 6자리 코드를 파트너에게 공유하여 그룹에 참여시킨다. 그룹 최대 인원은 5인이다.

**입력 / 출력**
- 입력 (그룹 생성): 그룹 이름 (선택, 미입력 시 기본값 "우리의 캘린더")
- 출력 (그룹 생성): 그룹 ID, 초대 링크(딥링크), 6자리 초대 코드 생성
- 입력 (참여): 초대 링크 클릭 또는 초대 코드 입력
- 출력 (참여): 그룹 캘린더 화면으로 즉시 진입, 그룹 생성자에게 참여 알림 발송

**예외 처리 및 엣지케이스**
- 이미 같은 그룹에 속한 사용자가 초대 코드를 재입력하면 "이미 그룹에 참여 중입니다" 안내
- 초대 링크/코드 유효 기간: 72시간 (만료 후 재발급 가능)
- 앱 미설치 상태에서 초대 링크 클릭 시 App Store로 이동 후 앱 설치 완료 시 자동으로 그룹 참여 처리 (Deferred Deep Link)
- 그룹 인원이 5인에 달한 경우 추가 초대 불가 안내
- 그룹 탈퇴 시 해당 사용자의 개인 일정만 유지, 공동 일정에서 제거

**우선순위**: 최우선 (P0)

---

### F03. 개인 캘린더 CRUD

**기능 설명 (사용자 관점)**
하루/주/월 뷰를 전환하며 일정을 조회한다. 일정을 탭하여 상세 조회하거나, 날짜를 길게 눌러 새 일정을 생성한다. 생성한 일정은 수정 및 삭제할 수 있다.

**입력 / 출력**
- 입력 (생성/수정): 제목(필수), 날짜/시간(필수), 종일 여부, 메모(선택), 공유 여부(개인/그룹)
- 출력 (생성): 캘린더에 일정 표시, 그룹 공유 선택 시 동기화 트리거
- 출력 (삭제): 캘린더에서 즉시 제거, 공유 일정인 경우 그룹 전원에게 삭제 알림

**예외 처리 및 엣지케이스**
- 제목 없이 저장 시도 시 인라인 에러 표시
- 종료 시각이 시작 시각보다 이른 경우 경고 후 저장 불가
- 오프라인 상태에서 생성/수정 시 로컬에 임시 저장, 온라인 복귀 시 자동 동기화
- 동일 시간대에 여러 일정이 겹치는 경우 세로 분할 레이아웃으로 표시
- 일정 삭제 시 "삭제하시겠습니까?" 확인 다이얼로그 표시 (실수 방지)
- 공유 일정을 개인 일정으로 변경하는 기능은 MVP에서 미지원 (변경 시 삭제 후 재생성 안내)

**우선순위**: 최우선 (P0)

---

### F04. 실시간 공유 동기화

**기능 설명 (사용자 관점)**
그룹 내 한 명이 일정을 생성, 수정, 삭제하면 다른 멤버의 캘린더에 즉시 반영된다. 앱을 다시 열지 않아도 변경 내용이 자동으로 갱신된다.

**입력 / 출력**
- 입력: 그룹 공유 설정이 적용된 일정의 생성/수정/삭제 이벤트
- 출력: Firebase Realtime Database 또는 Firestore를 통한 그룹 전원의 캘린더 즉시 갱신 (목표 지연 시간: 2초 이내)

**예외 처리 및 엣지케이스**
- 두 명이 동시에 같은 일정을 수정하는 경우: 마지막 저장 기준(Last Write Wins) 정책 적용, 덮어쓰기 발생 시 "다른 참여자가 수정했습니다. 최신 버전을 확인하세요" 알림
- 오프라인 상태에서 수정 후 온라인 복귀 시: 로컬 변경사항을 서버에 업로드하되, 서버 버전과 충돌이 있으면 사용자에게 어떤 버전을 유지할지 선택 옵션 제공
- 동기화 실패(네트워크 오류) 시 재시도 큐에 등록, 자동 재시도 (최대 3회)
- 동기화 상태 표시: 캘린더 상단에 "동기화 중" / "마지막 동기화: X분 전" 표시

**우선순위**: 최우선 (P0)

---

### F05. 공동 일정 초대 및 수락/거절

**기능 설명 (사용자 관점)**
일정 생성 시 또는 기존 일정에서 그룹 멤버를 초대한다. 초대받은 멤버는 푸시 알림을 통해 수락 또는 거절을 선택한다. 수락하면 해당 일정이 자신의 캘린더에 자동으로 추가된다.

**입력 / 출력**
- 입력: 초대 대상(그룹 멤버), 일정 정보
- 출력 (초대): 상대방에게 푸시 알림 발송, 초대 상태 "대기중" 표시
- 입력 (수락/거절): 수락 또는 거절 선택
- 출력 (수락): 수락자의 캘린더에 일정 추가, 초대자에게 수락 푸시 알림 발송
- 출력 (거절): 초대자에게 거절 푸시 알림 발송, 수락자 캘린더에 추가되지 않음

**예외 처리 및 엣지케이스**
- 상대방이 앱 알림 권한을 거부한 경우: 앱 진입 시 인박스(알림 센터)에서 초대 확인 가능
- 초대 수락/거절 응답 없이 48시간 경과 시: 자동으로 "미응답" 상태로 처리, 초대자에게 "아직 응답하지 않았어요" 알림
- 초대받은 일정이 자신의 기존 일정과 시간이 겹치는 경우: 수락 전 경고 메시지 표시 ("기존 일정과 시간이 겹칩니다. 그래도 수락하시겠습니까?")
- 초대 취소: 수락 전까지 초대자가 초대 취소 가능, 취소 시 상대방에게 알림
- 수락 후 일정 참여 취소: "참여 취소" 기능으로 캘린더에서 제거, 초대자에게 알림

**우선순위**: 최우선 (P0)

### F06. 푸시 알림

**기능 설명 (사용자 관점)**
공동 일정 초대, 초대 수락/거절, 일정 리마인더(당일/전날)를 푸시 알림으로 수신한다. 알림을 탭하면 해당 일정 상세 화면으로 바로 이동한다.

**입력 / 출력**
- 입력: 이벤트 트리거 (초대, 수락, 거절, 일정 임박)
- 출력: iOS 시스템 푸시 알림 발송 (APNs를 통한 FCM 연동)

**알림 유형 상세**

| 유형 | 발송 시점 | 알림 문구 예시 |
|------|-----------|---------------|
| 공동 일정 초대 | 초대 발생 즉시 | "{상대방 이름}님이 '{일정 제목}'에 초대했어요" |
| 초대 수락 | 수락 즉시 | "{상대방 이름}님이 '{일정 제목}'을 수락했어요" |
| 초대 거절 | 거절 즉시 | "{상대방 이름}님이 '{일정 제목}'을 거절했어요" |
| 일정 리마인더 (전날) | 해당 일정 전날 오전 9시 | "내일 '{일정 제목}'이 있어요" |
| 일정 리마인더 (당일) | 해당 일정 1시간 전 | "1시간 후 '{일정 제목}'이 시작돼요" |
| 일정 변경 알림 | 공유 일정 수정 즉시 | "{상대방 이름}님이 '{일정 제목}'을 수정했어요" |

**예외 처리 및 엣지케이스**
- 알림 권한 미부여 사용자: 앱 진입 시 인박스에서 확인 가능, 설정 화면에서 알림 권한 안내
- 앱 미설치 기기로의 초대: 알림 대신 초대 코드 링크 공유로 안내
- 알림 발송 실패(네트워크 등): FCM 재시도 메커니즘으로 자동 재전송 (최대 3회)
- 야간 방해 금지 모드 (오후 11시~오전 8시): 리마인더 알림은 해당 시간대에 발송 보류, 오전 8시에 일괄 발송. 단, 당일 리마인더는 즉시 발송

**우선순위**: 최우선 (P0)

### F07. 개인정보처리방침 및 이용약관

**기능 설명 (사용자 관점)**
앱 최초 실행 또는 회원가입 시 개인정보처리방침 및 이용약관 동의 화면이 표시된다. 전문은 앱 내 웹뷰 또는 외부 링크로 열람 가능하다. 설정 메뉴에서 언제든 재확인할 수 있다.

**입력 / 출력**
- 입력: 필수 동의 체크 및 "동의 후 시작하기" 버튼 탭
- 출력: 동의 기록 저장 (동의 일시 및 버전 포함), 온보딩 진행

**예외 처리 및 엣지케이스**
- 필수 동의 미체크 상태에서 진행 시도 시 인라인 안내 메시지 표시
- 개인정보처리방침 업데이트 시 기존 사용자에게 재동의 요청

**우선순위**: 최우선 (P0) — 앱스토어 심사 필수 요건

### F08. 구글 캘린더 연동 (v1.1)

**기능 설명 (사용자 관점)**
설정에서 Google 계정을 연결하면 Google Calendar의 일정이 WeCal에 표시된다. WeCal에서 생성한 일정은 선택적으로 Google Calendar에도 동기화할 수 있다.

**입력 / 출력**
- 입력: Google OAuth 2.0 인증 (캘린더 읽기/쓰기 Scope)
- 출력: Google Calendar 이벤트를 WeCal 캘린더에 레이어로 표시 (색상 구분)

**예외 처리 및 엣지케이스**
- OAuth 인증 실패 시 재인증 안내
- Google API Rate Limit 초과 시 "잠시 후 다시 시도해주세요" 안내
- Google 토큰 만료 시 자동 갱신 (리프레시 토큰 활용)
- Google Calendar와 WeCal 양쪽에 동일 일정 중복 생성 방지

**우선순위**: 높음 (P1) — v1.1 대상, Google OAuth 앱 검토 승인 후 출시

### 5-1. 기술적 제약

## 6. 오픈 이슈

| 번호 | 항목 | 내용 | 담당 |
