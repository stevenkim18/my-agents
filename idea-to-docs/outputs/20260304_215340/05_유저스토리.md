## 2. 유저 스토리

### F01. 회원가입 및 로그인

---

```
스토리 ID: US-01
페르소나: 지수, 하은, 유진 (신규 사용자 전체)
스토리: 나는 새로운 앱 사용자로서, 복잡한 회원가입 절차 없이 빠르게 시작하기 위해
        Sign in with Apple로 간편하게 로그인하고 싶다.

인수 조건 (Acceptance Criteria):
- Given: 앱을 처음 실행한 상태
- When: "Apple로 로그인" 버튼을 탭한다
- Then: Apple ID 인증이 완료되고 계정이 생성되어 온보딩 화면으로 이동한다

우선순위: Must Have
예상 포인트: 3
```

```
스토리 ID: US-02

스토리: 나는 기존 사용자로서, 앱을 열 때마다 다시 로그인하지 않기 위해
        자동 로그인 상태가 유지되길 원한다.

인수 조건 (Acceptance Criteria):
- Given: 이전에 로그인한 상태에서 앱을 종료하고 다시 실행한다
- When: 앱이 실행된다
- Then: 별도의 인증 절차 없이 캘린더 메인 화면으로 바로 진입한다

우선순위: Must Have
예상 포인트: 2
```

```
스토리 ID: US-03
페르소나: 지수, 하은, 유진 (기기 교체 사용자)
스토리: 나는 새 iPhone으로 기기를 교체한 사용자로서, 기존 데이터를 유지하기 위해
        동일한 Apple ID로 로그인 시 기존 계정과 일정이 복원되길 원한다.

- Given: 새 기기에서 앱을 설치하고 기존에 사용하던 Apple ID로 로그인을 시도한다

- Then: 기존 그룹, 일정, 설정 정보가 모두 복원되어 캘린더 메인 화면으로 이동한다

우선순위: Must Have
예상 포인트: 3
```

### F02. 그룹 생성 및 초대

---

```
스토리 ID: US-04
페르소나: 민준 (그룹 생성자)
스토리: 나는 파트너와 처음 WeCal을 시작하는 사용자로서, 우리만의 공유 공간을 만들기 위해
        그룹을 생성하고 파트너에게 초대 링크를 보내고 싶다.

인수 조건 (Acceptance Criteria):
- Given: 온보딩 또는 설정 화면에서 그룹 생성 버튼을 탭한다
- When: 그룹 이름을 입력(또는 기본값 유지)하고 생성을 완료한다
- Then: 고유한 초대 링크(딥링크)와 6자리 초대 코드가 생성되어 공유 가능한 상태가 된다

우선순위: Must Have
예상 포인트: 3
```

---

```
스토리 ID: US-05
페르소나: 지수 (초대받은 파트너)
스토리: 나는 파트너로부터 초대를 받은 사용자로서, 간편하게 그룹에 합류하기 위해
        6자리 초대 코드를 입력하거나 초대 링크를 탭하여 그룹에 참여하고 싶다.

인수 조건 (Acceptance Criteria):
- Given: 파트너에게 초대 코드 또는 링크를 전달받은 상태
- When: 앱에서 초대 코드를 입력하거나 초대 링크를 탭한다
- Then: 그룹에 즉시 합류되어 공유 캘린더 화면으로 이동하고, 그룹 생성자에게 참여 알림이 발송된다

우선순위: Must Have
예상 포인트: 3
```

```
스토리 ID: US-06

- Then: 새 멤버가 코드/링크로 그룹에 참여할 수 있고, 기존 멤버 전원에게 참여 알림이 발송된다

우선순위: Must Have
예상 포인트: 2
```

### F03. 개인 캘린더 CRUD

---

```
스토리 ID: US-07
페르소나: 지수 (일정 생성자)
스토리: 나는 바쁜 직장인으로서, 새로운 약속이 생겼을 때 빠르게 기록하기 위해
        캘린더에서 날짜를 길게 눌러 일정을 빠르게 생성하고 싶다.

인수 조건 (Acceptance Criteria):
- Given: 캘린더 메인 화면(월/주/일 뷰)에서 특정 날짜를 길게 누른다
- When: 일정 생성 폼이 열리고 제목, 날짜/시간을 입력하고 저장한다
- Then: 해당 날짜에 일정이 즉시 표시되고, 그룹 공유를 선택한 경우 파트너 캘린더에도 동기화된다

우선순위: Must Have
예상 포인트: 5
```

---

```
스토리 ID: US-08
페르소나: 민준 (일정 수정자)
스토리: 나는 갑자기 일정이 변경된 사용자로서, 파트너에게 별도 연락 없이 변경사항을 알리기 위해
        기존 일정을 수정하고 파트너가 즉시 알 수 있도록 하고 싶다.

인수 조건 (Acceptance Criteria):
- Given: 캘린더에 이미 생성된 공유 일정이 있는 상태
- When: 해당 일정을 탭하여 상세 화면에서 시간/내용을 수정하고 저장한다
- Then: 수정된 내용이 내 캘린더에 즉시 반영되고, 파트너에게 "일정이 변경되었습니다" 푸시 알림이 발송된다

우선순위: Must Have
예상 포인트: 3
```

```
스토리 ID: US-09

우선순위: Must Have
예상 포인트: 2
```

```
우선순위: Must Have
예상 포인트: 3
```

### F04. 실시간 공유 동기화

---

```
스토리 ID: US-11
페르소나: 민준 (일정 변경 수신자)
스토리: 나는 파트너의 일정 변경을 놓치지 않으려는 사용자로서, 지수가 일정을 수정했을 때

- Then: 2초 이내에 내 캘린더에 변경 내용이 자동으로 반영된다

우선순위: Must Have
예상 포인트: 5
```

```
- Then: 로컬 캐시로 기존 일정을 조회할 수 있고, 신규 생성/수정 내용은 임시 저장되어 온라인 복귀 시 자동으로 동기화된다
```

### F05. 공동 일정 초대 및 수락/거절

---

```
스토리 ID: US-13
페르소나: 준서 (공동 일정 생성자)
스토리: 나는 서울-부산 장거리 연애 중인 사용자로서, 다음 만남 날짜를 확정하기 위해
        공동 일정을 생성하고 하은에게 참석 초대를 보내고 싶다.

인수 조건 (Acceptance Criteria):
- Given: 일정 생성 폼에서 날짜, 시간, 제목을 입력한 상태
- When: 그룹 멤버(하은)를 초대 대상으로 선택하고 일정을 저장한다
- Then: 하은에게 "공동 일정 초대" 푸시 알림이 발송되고, 내 캘린더에는 해당 일정이 "대기중" 상태로 표시된다

우선순위: Must Have
예상 포인트: 3
```

---

```
스토리 ID: US-14
페르소나: 하은 (초대 수신자 — 수락)
스토리: 나는 준서로부터 공동 일정 초대를 받은 사용자로서, 빠르게 참석 여부를 확정하기 위해
        푸시 알림에서 바로 수락 또는 거절을 선택하고 싶다.

인수 조건 (Acceptance Criteria):
- Given: "공동 일정 초대" 푸시 알림을 받은 상태
- When: 알림을 탭하여 일정 상세 화면으로 이동한 뒤 "수락" 버튼을 탭한다
- Then: 해당 일정이 내 캘린더에 자동으로 추가되고, 준서에게 "수락" 푸시 알림이 발송된다

우선순위: Must Have
예상 포인트: 3
```

---

```
스토리 ID: US-15
페르소나: 하은 (초대 수신자 — 거절)
스토리: 나는 이미 교대 근무가 잡혀 있는 날 초대를 받은 사용자로서, 참석이 불가능한 경우
        거절 의사를 간편하게 전달하고 싶다.

인수 조건 (Acceptance Criteria):
- Given: 공동 일정 초대 상세 화면이 열려 있는 상태
- When: "거절" 버튼을 탭한다
- Then: 내 캘린더에 해당 일정이 추가되지 않고, 준서에게 "거절" 푸시 알림이 발송된다

우선순위: Must Have
예상 포인트: 2
```

---

```
스토리 ID: US-16
페르소나: 상훈 (수락 후 참여 취소자)
스토리: 나는 이미 수락한 일정에 갑작스러운 외근이 생긴 사용자로서, 파트너에게 상황을 알리기 위해
        수락한 공동 일정 참여를 취소하고 싶다.

인수 조건 (Acceptance Criteria):
- Given: 수락한 공동 일정의 상세 화면이 열려 있는 상태
- When: "참여 취소" 버튼을 탭하고 확인한다
- Then: 내 캘린더에서 해당 일정이 제거되고, 초대자(유진)에게 "참여를 취소했어요" 알림이 발송된다

우선순위: Must Have
예상 포인트: 2
```

---

```
스토리 ID: US-17
페르소나: 유진 (초대 취소자)
스토리: 나는 일정이 취소된 경우, 상대방이 수락하기 전에 초대를 철회하기 위해
```

## 4. 유저 여정 맵 (User Journey Map)

신규 사용자(지수)가 처음 WeCal을 설치하고 파트너(민준)와 공동 일정을 경험하기까지의 흐름입니다.

### 단계 1 — 앱 발견 및 설치

- **행동**: 지수가 친구 추천으로 App Store에서 WeCal을 검색하고 설치한다.
- **생각**: "카카오톡으로 일정 조율하는 게 너무 귀찮아. 이거 쓰면 좀 편해질까?"
- **감정**: 기대 반, 의심 반
- **터치포인트**: App Store 설명 페이지, 스크린샷, 리뷰

### 단계 2 — 온보딩 및 회원가입

- **행동**: 앱을 열면 스플래시 화면 이후 이용약관 동의 화면이 표시된다. 필수 항목에 동의하고 Sign in with Apple 버튼을 탭한다.
- **생각**: "이거 진짜 간단하네. Apple 계정으로 바로 되네."
- **감정**: 안도 (복잡한 회원가입이 없다는 점)
- **터치포인트**: 동의 화면, Sign in with Apple 버튼

### 단계 3 — 그룹 생성 및 파트너 초대

- **행동**: 온보딩 화면에서 "파트너 초대하기" 버튼을 탭하고 그룹을 생성한다. 생성된 6자리 초대 코드를 카카오톡으로 민준에게 공유한다.
- **생각**: "코드가 생겼네. 민준이한테 보내면 되는 거잖아."
- **감정**: 쉽다는 안도감, 민준이 수락할지에 대한 기대감
- **터치포인트**: 그룹 생성 화면, 초대 코드 공유 시트

### 단계 4 — 파트너의 그룹 참여

- **행동**: 민준이 카카오톡 링크를 클릭하고 앱을 설치한다. 코드를 입력하거나 링크를 통해 그룹에 자동으로 참여한다.
- **지수의 경험**: "민준이 합류했어요" 푸시 알림을 받는다.
- **감정**: 성공적으로 연결된 것에 대한 만족감
- **터치포인트**: 딥링크, 코드 입력 화면, 참여 알림

### 단계 5 — 첫 번째 공유 일정 생성

- **행동**: 지수가 이번 주 토요일 저녁 약속을 공유 일정으로 등록한다. 민준을 공동 참석자로 초대한다.
- **생각**: "그냥 여기서 일정 만들면 민준이도 보이는 거잖아."
- **감정**: 신기함, 편리함에 대한 첫 체험
- **터치포인트**: 일정 생성 폼, 그룹 멤버 초대 선택 화면

### 단계 6 — 파트너의 수락 및 상호 확인

- **행동**: 민준이 "공동 일정 초대" 알림을 받고 탭하여 상세 화면에서 수락한다. 지수에게 "수락" 알림이 전달된다.
- **지수의 경험**: "민준이 토요일 저녁 약속을 수락했어요" 알림을 받고 캘린더에서 두 사람 모두의 일정이 반영된 것을 확인한다.
- **감정**: WeCal의 핵심 가치를 처음으로 실감하는 순간 — "이게 바로 내가 원하던 거야!"
- **터치포인트**: 푸시 알림, 일정 상세 화면, 공유 캘린더 뷰

### 단계 7 — 일정 변경 및 알림 경험

- **행동**: 지수에게 갑자기 야근이 생겨 토요일 저녁 약속 시간을 변경한다. WeCal에서 일정을 수정한다.
- **민준의 경험**: "지수님이 '토요일 저녁 약속'을 수정했어요" 알림을 즉시 수신하고, 별도 연락 없이 변경 내용을 확인한다.
- **감정**: 두 사람 모두 "이제 카카오톡으로 따로 알릴 필요가 없겠다"는 핵심 가치 확인
- **터치포인트**: 일정 수정 화면, 변경 푸시 알림, 상대방 캘린더 자동 반영
